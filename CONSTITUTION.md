# Constitution — AI UI Testing Tool

The rules this project will **never break**, regardless of time pressure or
convenience. Every agent and stage must respect these.

Keep this list short — 3 to 7 rules. More than that and nothing is truly
non-negotiable.

---

## Rules

1. **The Spec is the contract.** If it isn't in the Spec, it doesn't get built. Scope changes go through the Spec first.
2. **Nothing ships unverified.** Every feature is validated against its acceptance criteria before it's considered done.
3. **Keep it simple.** Favor the simplest approach that satisfies the Spec; no premature abstraction or speculative generality.
4. **Determinism over flakiness.** A UI test that passes and fails randomly is a bug — reliability is a first-class requirement.
